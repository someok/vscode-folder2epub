# vscode-folder2epub Task 文档

## 1. 项目目标

初始化一个 VS Code 扩展项目，实现针对“符合条件的目录”的右键菜单能力，并为后续 EPUB 相关功能预留清晰的实现路径。

目标菜单：

- 生成 epub
- 初始化 epub

说明：

- 菜单应出现在 VS Code 资源管理器中，对目录节点右键时可见。
- 菜单只对“符合条件的目录”显示。
- 当前阶段先完成任务拆解与实现约束文档，后续按本文档逐步落地。

符合条件的目录定义：
- 右键选定的目录中存在子目录 `__t2e.data`，且其中存在 yaml 文件 `metadata.yml`

### 3.2 “初始化 epub”命令预期行为

当前状态：`已定义`

初始化 epub 行为：
- 创建文件夹 `__t2e.data`
- 在 `__t2e.data` 下创建 yaml 文件 `metadata.yml`
- 在 `metadata.yml` 中写入默认内容：
  ```yaml
  title: [默认为当前选定的文件夹]
  titleSuffix: ''
  author: i3166
  description: ''
  cover: cover.jpg
  version: 1.0.0
  ```
- 覆盖策略：
  - 如果已经存在 `metadata.yml` 则提示该文件已存在，然后放弃操作

### 3.3 “生成 epub”命令预期行为

当前状态：`已定义`

生成 epub 行为：
- 如果右键选定的文件夹下不存在 `__t2e.data/metadata.yml` 则此菜单为 disabled 状态
- 如果右键选定的文件夹中没有除 `__t2e.data` 外的其它子文件夹或 md、txt 文件，则抛出提示，之后放弃操作
- 支持生成 epub3 格式的 epub
- 文件来源支持 txt 和 markdown
  - txt 转换为 epub 要求的 HTML
  - markdown 转换为 epub 要求的 HTML
- 生成的 epub 文件名格式为：《TITLE》SUFFIX作者_AUTHOR.epub
  - TITLE: metadata.yml 中的 title 字段对应内容
  - SUFFIX：metadata.yml 中的 titleSuffix 字段对应内容，如果 titleSuffix 不为空，则使用中文括号括起来
  - AUTHOR：metadata.yml 中的 author 字段对应内容，如果此字段为空，则默认为「佚名」
  - 示例：
    - 示例 `metadata.yml`
      ```yaml
      title: 测试文档
      titleSuffix: 待续
      author: i3166
      description: ''
      cover: cover.jpg
      version: 1.0.0
      ```
    - 生成的 epub 为：《测试问答》（待续）作者_i3166.epub
- 生成的 epub 文件位置
  - 检测当前文件夹所有父目录，只要找到 `__epub.yml` 且其中配置了 `saveTo` 字段（不为空），则使用此 `saveTo` 定义的文件夹
  - 如果所有父目录均没有 `__epub.yml` 和 `saveTo` 配置，则默认将 epub 文件输出到当前选定的文件夹中
- 生成 epub 的文件来源
  - 选定文件夹下（包含子文件）txt、md 文件
  - 子文件夹和文件如果有数字前缀，使用单个或多个下划线分割，则表示使用该数字排序，例如 `0120_实际的章节名.md`
    - 实际的文件夹或文件名称为去除前缀之后的内容
  - 如果没有数字前缀，则直接使用文件名排序
  - 如果有子文件（非 `__t2e.data`），则子文件名（去除前缀）为 epub 中的目录，且目录指向的是子文件夹中的第一个文件
    - 子文件夹可能有多层

## 7. 风险与实现备注

### 已知风险

- VS Code 菜单的“只对符合条件目录显示”能力，可能不能完全靠静态 `when` 子句完成。
- 如果“符合条件”依赖目录内部文件内容，通常需要命令执行时再次校验。
- 如果要实现较强的动态显示控制，可能需要引入额外上下文同步机制。

### 建议实现原则

- 先做最小可运行版本，再做条件过滤增强。
- 先完成命令注册与菜单出现，再补充真实 EPUB 逻辑。
- 显示条件和执行条件同时做，避免仅靠 UI 控制导致误操作。
